Dit document beschrijft een applicatieprofiel, in dit geval Verkeersbordendatabank. Het applicatieprofiel is ontwikkeld binnen het project Lokale Besluiten als Gelinkte Open Data en laat toe om de verkeersbordendatabank gelinkt te ontsluiten zodat besluiten van lokale besturen hiermee kunnen linken. Het beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Samenvatting

Dit applicatieprofiel beschrijft een informatiemodel om opstellingen van verkeersborden op het openbaar domein gelinkt op te stellen en te delen. Enerzijds beschrijft dit model basisentiteiten (Opstelling, Steun, Bevestiging en Verankering) om het fysieke aspect van een verkeersbordopstelling te beschrijven, anderzijds wordt er dieper ingegaan op de betekenis van dit fysieke verkeersbord met de entiteit Verkeerbordbasis. Wat er beslist wordt in een lokaal besluit slaat op deze laatste. Verkeersbord en Verkeersbordbasis zijn opgesplitst, omdat een fysiek verkeersbord meerdere verkeersborden kan bevatten op hetzelfde vlak. Verkeersbordbasis slaat hier dus op zijn conceptuele vorm en Verkeersbord op de fysieke vorm zoals aangegeven in OSLO-OpenbaarDomein. Dit model stelt voor om een gecontroleerde lijst op te bouwen van verkeersborden. Verkeersbord 'F1' is bijvoorbeeld een concept. Sommige borden, zoals wegwijzers (bv.: F29), vragen een specifieke invulling. Verkeersbordbasis stelt hierbij de lokale implementatie voor van dit soort verkeersbord met als 'variabel opschrift' wat er op het bord komt.

Status van dit document

Dit applicatieprofiel dient om het specifieke gebruik van de entiteiten relevant voor de beschreven applicatie te verduidelijken.

Dit document werd gepubliceerd als Ontwerpdocument. De specificatie zoals in dit document beschreven is in ontwikkeling. Bijgevolg kan dit document op elk moment aangepast, vervangen of achterhaald worden.

Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • verfijning van de terminologie (klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.

Om conform te zijn met dit applicatieprofiel, geldt voor een implementatie dat ze:

  • MOET Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • MAG NIET meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • MAG terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • MAG NIET terminologie van andere gecontroleerde vocabularia gebruiken dan diegene die gedefinieerd wordt in dit applicatieprofiel.
  • MAG uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

Entiteiten

Artikel wegcode

Beschrijving
Een deel van een rechtsgrond,op een arbitrair niveau van precisie,zoals een hoofdstuk,artikel,paragraaf of lijst element.
Gebruik
Bijvoorbeeld: artikel 12.1 Verkeersborden F1, F1a of F1b...
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

Bevestiging

Beschrijving
Het montage-element dat een verkeersbord bevestigt aan een infrastructuurelement.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: conform standaard, montagetype.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
conform standaard String 1 Tot welke standaard deze bevestiging conform is.
montagetype Code 1 Waarmee de bevestiging gemonteerd is.

Infrastructuurelement

Beschrijving
Individueel object op het Openbaar Domein dat niet vegetatief is.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: beheerder, breedte, diameter, fabricagevoorschrift, kleur, lengte bovengronds, lengte ondergronds, materiaal, plaatsingsdatum, verankerd met, verwijderingsdatum.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beheerder Organisatie 1 Instantie (tot op team- of dienstniveau) die instaat voor het beheer van het object.
breedte GetypeerdeString 1 Afmeting van het infrastructuurelement. Bij verankering: de afmeting loodrecht op de lengte-as.
diameter GetypeerdeString 1 Doorsnede van het infrastructuur element. Bij verankering: doorsnede van de verankering.
fabricagevoorschrift String 1 Genormaliseerde referentie waaraan het infrastructuur element aan voldoet.
kleur Code 1 Kleur van het infrastructuur element.
lengte bovengronds Integer 1 Lengte van het infrastructuur element boven het verhardingsniveau of maaiveld.
lengte ondergronds Integer 1 Lengte van het infrastructuur element onder het verhardingsniveau of maaiveld.
materiaal Materialen 1 Materiaal waar het object uit vervaardigd is.
plaatsingsdatum Datetime 1 Datum waarop het infrastructuur element geïnstalleerd is.
verankerd met Verankering 0...1 Waarmee het infrastructuurelement verankerd is.
verwijderingsdatum Datetime 1 Datum waarop het infrastructuur element verwijderd is.

Beschrijving

Eigenschappen

omvat infrastructuurelement Infrastructuurelement 0...* Infrastructuurelement die voor de ondersteuning zorgt in de opstelling.
omvat verkeersbord Verkeersbord 1...* Verkeersbord die op de opstelling staat.
plaats Plaats 1 Locatie van de opstelling.
positie ten opzichte van rijweg Code 1 Manier waarop de opstelling zich verhoudt tot de rijbaan.
status Code 1 Status van de opstelling.

Verkeersbord

Beschrijving
Wegsignalisatie aangebracht op een element op het openbaar domein (paal, gevel, portiek...) om het verkeer op de openbare weg te regelen en die geen verkeerslicht zijn.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanzicht, afbeelding, afmeting, bevestigd met, folietype, opgehangen met, opmerking, opstelhoogte, realiseert.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aanzicht GetypeerdeString 1 De hoek waarin het fysiek bord gepositioneerd is ten opzichte van een vooropgestelde as.
afbeelding Document 1 Weergave van het geïnstalleerde verkeersbord op het openbaar domein.
afmeting Code 1 Afmeting (lengte) van het verkeersbord.
bevestigd met Bevestiging 0...* Waarmee het verkeersbord is bevestigd aan een infrastructuurelement.
folietype Code 1 Type van de folie die bevestigd is aan het verkeersbord.
opgehangen met Infrastructuurelement 0...* Waaraan het verkeersbord is opgehangen.
opmerking String 1 Opmerking over het verkeersbord.
opstelhoogte GetypeerdeString 1 Afstand tussen het maaiveld en de onderrand van het bord.
realiseert Verkeersbordbasis 1...* Relateert een verkeersbord op het openbaar domein tot de verkeersbordbasis gerealiseerd door het (fysieke) verkeersbord.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: adresvoorstelling, afbeelding, heeft gerelateerd bord, heeft onderbord, heeft verkeersbordconcept, is zonebord, plaatsbeschrijving, status, variabel opschrift.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

afbeelding Document 1 Foto van de verkeersbordbasis.
heeft gerelateerd bord Verkeersbordbasis Verkeersbordbasis die hieraan gerelateerd is.
heeft onderbord Verkeersbordbasis 0...* Onderbord die behoort tot dit verkeersbordbasis.
heeft verkeersbordconcept Verkeersbordconcept 1 Algemeen verkeersbordconcept van de verkeersbordbasis.
